# UI 핸드오프 — 프롬프트 중간고사 평가 대시보드

> **목적**: 이 문서는 Claude Design 시스템 단계가 받아갈 입력입니다. 기능/아키텍처는 [`../superpowers/specs/2026-04-29-prompt-eval-dashboard-design.md`](../superpowers/specs/2026-04-29-prompt-eval-dashboard-design.md) 참조.
>
> 여기서는 **UI가 무엇을 느끼게 해야 하는지**, **어떤 순간이 핵심인지**, **어떤 정보가 어디에 위치해야 하는지** 만 정의합니다. 색/타이포/외형은 디자인 시스템에서 결정.

---

## 1. 제품 톤 & 무드

### 한 줄 정체성
**"노래방 점수 공개 × 캐릭터 챗봇 × 라이브 평가"**

학생이 발표를 마치고 모두가 화면을 응시하는 순간, 챗봇이 통통 튀어나와 짖궂게 질문을 던지고, 점수가 한 줄씩 드라마틱하게 채워지다가 마지막에 합계가 빵— 터지는 경험.

### 톤 키워드
- **긴장감 (Tension)** — 발표가 끝난 직후의 두근거림, "내 점수 몇 점 나올까?"
- **장난스러움 (Playful)** — 챗봇 캐릭터가 분위기를 풀어줌. 너무 진지하지 않게.
- **드라마틱 (Dramatic)** — 점수 공개는 무대처럼. 숫자 하나가 천천히 차오름.
- **귀여움 (Cute)** — 챗봇 캐릭터는 보호 욕구를 자극할 정도로 사랑스러워야 함.

### 절대 피해야 할 것
- **사무용 BI 대시보드 같은 인상** (회색 표, 밋밋한 카드)
- **시험 채점 사이트 같은 차가움** (성적표 분위기)
- **AI 챗봇 흔한 말풍선 인터페이스**

## 2. 핵심 UI 모멘트 ("디자인이 빛나야 하는 순간")

이 5개 모멘트를 디자인이 살리느냐가 제품 성패를 가름.

### 🎤 모멘트 1. 챗봇 등장
- 챗봇이 화면 어딘가에서 통통 튀어 들어옴 (이전 단계 화면이 살짝 어두워지거나 뒤로 빠짐)
- 캐릭터가 살아있는 듯한 미세한 idle 애니메이션 (호흡, 눈 깜빡임 등)
- 학생/관객의 시선을 잡아끄는 첫 순간

### 💬 모멘트 2. 챗봇 질문 출제
- 챗봇이 말풍선으로 질문 한 줄을 띄움 (타이핑 애니메이션 권장)
- TA가 학생 답변을 받아치는 입력창은 챗봇과 한 묶음으로 시각 결합
- "다음 질문" 으로 넘어갈 때 챗봇이 살짝 점프

### ⏳ 모멘트 3. 채점 중 (5개 툴 동시 실행)
- "지금 AI가 채점 중" 임을 시각적으로 강하게 전달
- 5개 항목이 미리 5칸으로 보이고, 각 칸이 로딩 중인 상태로 펄스
- 챗봇이 두근두근 떨고 있거나 두 손 모으고 있는 포즈

### 🥁 모멘트 4. 하위 점수 순차 공개
- 5개 점수 카드가 **도착 순으로 (병렬 채점이라 순서 비결정적)** 한 장씩 뒤집히듯/올라오듯/펑 터지듯 공개
- 각 카드: 항목명 + 점수 (예: "27 / 30") + 평가 사유 한 줄
- 카드 간 최소 1.5초 간격 (너무 빠르면 긴장감 없음)

### 🏆 모멘트 5. 합계 대공개
- 5개 카드가 다 노출된 후, **최상단에 큰 합계 숫자**가 0부터 차올라 최종값에 도달
- 노래방 점수처럼 "툭—툭—툭" 한 자리씩 올라가도 좋고, 슬롯머신처럼 굴러도 좋음
- 클라이맥스: 챗봇이 점프하며 박수치는 식의 마무리 액션

## 3. 화면 상태별 정보 위계

머신 상태 6개. 각 상태에서 **무엇이 가장 중요한 정보인지** 만 명시.

### State A. `idle` (대기)
- **포커스**: "다음 학생 평가 시작" CTA
- **배경**: 챗봇이 idle 애니메이션으로 분위기 띄움
- **부속**: 헤더에 설정 버튼(API Key), 캐시 무시 토글

### State B. `input` (제출물 입력)
- **포커스**: 학생 이름, v1 prompt/result (필수 필드)
- **2차**: v2/v3 (옵션, 접혀 있다가 펼침), changeNote
- **분리 강조**: **프롬프트와 결과물은 명확히 다른 시각 구획**으로
- **CTA**: "평가 시작" (모든 필수 필드 채워졌을 때만 활성)
- **챗봇**: 한쪽 구석에서 "어떤 학생을 평가할까요?" 같은 멘트, 비주도적

### State C. `qa` (챗봇 Q&A)
- **포커스**: 챗봇 + 현재 질문 + 학생 답변 입력창
- **챗봇이 화면의 주인공**. 입력 폼은 시야에서 사라지거나 축소
- **진행도 표시**: 1/3 → 2/3 → 3/3 (비주얼 인디케이터, 숫자만이 아니라)
- **답변 입력 후 "다음" 클릭 시** 챗봇이 점프 → 다음 질문 등장

### State D. `grading` (채점 중)
- **포커스**: "지금 채점 중" 임을 보여주는 대기 화면
- **5개 항목 자리**가 미리 5개 placeholder로 깔려 있고 각자 로딩 펄스
- **챗봇**: 두근거리는 포즈
- **시간 감각**: 보통 5~20초. 너무 짧아도 긴장감 죽고, 너무 길어도 지루함 — 최소 ~3초는 보장 (이미 결과 와도 일부러 텀)

### State E. `reveal` (점수 공개)
- **포커스 1단계 (하위 점수 도착 시점)**: 방금 도착한 카드
- **포커스 2단계 (모두 도착 후)**: 최상단 합계 숫자
- **레이아웃**:
  ```
  ┌─ 합계 (가장 큼, 처음엔 ?? 또는 0) ─┐
  │                                       │
  ├─ 5개 하위 카드 (도착 순으로 채워짐) ─┤
  │  [설계] [출력] [반복] [발표] [창의]   │
  └───────────────────────────────────────┘
  ```
- **카드 내부**: 항목명, 점수 (숫자 큼), 만점 표시, 평가 사유 (작게)
- **합계 등장 타이밍**: 5번째 하위 카드가 안착한 직후

### State F. `done` (완료)
- **포커스**: 최종 점수판 + "다음 학생" CTA
- **챗봇**: "수고하셨어요!" 마무리 멘트
- **부속**: 평가 사유 펼쳐 보기, 다시 공개하기 (replay)

## 4. 컴포넌트 인벤토리

디자인 시스템에서 만들어야 할 핵심 컴포넌트들.

### 캐릭터
- **`<Chatbot>`** — 통통 튀는 마스코트. idle / talking / thinking / cheering / jumping 5개 포즈 최소.

### 입력
- **`<StudentNameInput>`** — 학생 이름 (싱글 라인)
- **`<PromptVersionCard>`** — 라벨(v1/v2/v3) + prompt 영역 + result 영역 + (옵션)changeNote. **프롬프트와 결과물 시각 분리 필수**.
- **`<AnswerInput>`** — TA가 학생 답변 받아치는 멀티라인 (단축키 Enter로 다음)

### 진행
- **`<QuestionBubble>`** — 챗봇이 던지는 질문 말풍선
- **`<ProgressIndicator>`** — Q&A 1/3, 2/3, 3/3
- **`<GradingPlaceholder>`** — 채점 중 5개 칸 펄스 로딩

### 점수
- **`<ScoreCard>`** — 항목명 + 점수 + 만점 + 평가 사유. 등장 애니메이션 필수.
- **`<TotalScoreReveal>`** — 합계 숫자 카운트업/슬롯 굴러가기 애니메이션

### 오버레이
- **`<SettingsModal>`** — API Key 입력
- **`<Toast>`** — 에러/안내

### 컨트롤
- **`<NextButton>`** — 단계별 진행 CTA (상태별로 라벨 바뀜)
- **`<CacheBypassToggle>`** — "캐시 무시하고 재평가"

## 5. 모션 랭귀지

### 키 모션 원칙
- **챗봇**: 살아있다는 느낌. 절대 정적이지 않음. spring 기반 통통 모션.
- **카드 등장**: scale-up + fade + 약간의 over-shoot (탄성)
- **숫자 카운트**: ease-out, 마지막 한 자리에 살짝 멈칫
- **상태 전환**: 직선적 fade 금지 → 챗봇이 다음 단계를 안내하듯 움직이며 자연스럽게 전환

### 타이밍 가이드
| 동작 | 권장 duration |
|---|---|
| 카드 등장 (개별) | 400~600ms |
| 카드 간 간격 | 1500ms 이상 |
| 합계 카운트업 | 1200~2000ms |
| 챗봇 점프 | 300~500ms |
| 모달 오픈 | 200~300ms |

## 6. 정보 분리 — 프롬프트 ↔ 결과물

**기능 요구사항**: "학생들이 제출한 결과물 (프롬프트 + 프롬프트 사용 결과) 를 넣고 (이건 인터페이스상 분리가 되면 좋겠음)"

### 입력 화면에서
- 같은 카드 안에서도 prompt 영역 / result 영역이 **시각적으로 명확히 구분**돼야 함
- 가능한 패턴: 좌우 split, 상하 분할, 탭, 색상/배경 차이
- 학생이 무엇을 입력했는지(prompt) vs 무엇이 나왔는지(result) 한눈에 구분

### 평가 결과 화면에서
- 평가 카드의 reasoning 텍스트가 prompt를 평가했는지 / result를 평가했는지 알 수 있도록
- 항목명만으로 충분히 분간되긴 함 (설계품질=프롬프트, 출력품질=결과)

## 7. 레이아웃 존

```
┌──────────────────────────────────────────────┐
│  [헤더: 타이틀 / 설정 / 캐시 토글]           │
├──────────────────────────────────────────────┤
│                                                │
│  [메인 캔버스]                                 │
│   - 상태에 따라 input/qa/grading/reveal 전환  │
│                                                │
│  [챗봇 영역]                                   │
│   - 화면의 한 코너 또는 메인에 통합           │
│   - 상태에 따라 위치/크기 변화 가능           │
│                                                │
└──────────────────────────────────────────────┘
                                       (Modal: SettingsModal, Toast)
```

- **단일 페이지**, 라우팅 없음
- **풀스크린에 가까운 캔버스**. 라이브 시연 환경이라 화면이 영사기로 비춰질 수 있음 → 큰 텍스트, 큰 숫자.

## 8. 디자인 시스템에 결정 위임하는 항목

이 핸드오프에서는 **정의하지 않음**. 디자인 시스템 단계에서 결정.

- 색상 팔레트 (주조색, 강조색, 배경)
- 타이포그래피 (한글/영문 폰트, 점수 숫자용 디스플레이 폰트)
- 챗봇 캐릭터의 구체적 외형 (모양, 표정, 색)
- 카드 스타일 (그림자, 라운드, 보더)
- 다크/라이트 모드 여부
- 일러스트레이션 스타일
- 사운드 (점수 공개 시 효과음 등) — 필요 여부 자체가 디자인 결정사항

## 9. 환경 제약

- **타깃 디바이스**: TA 노트북 → 프로젝터/대형 모니터로 미러링 가능성 ↑
- **해상도**: 1280×720 이상. 풀HD에서 가장 잘 보여야 함.
- **모바일/태블릿**: 비대상
- **다크 모드**: 라이브 발표 환경(조명 어두울 수 있음) 감안하면 다크 또는 다크 친화적 디자인이 자연스러움 — 디자인 시스템에서 판단

## 10. 영감 / 레퍼런스 키워드

디자인 시스템 단계에서 무드보드 만들 때 출발점:

- **노래방 점수 공개** (한국식, 90~100점 화려한 폭죽)
- **K-pop 데몬 헌터스 캐릭터** 같은 통통 튀는 마스코트
- **Apple Keynote 의 카운트업 애니메이션**
- **Duolingo 의 캐릭터 인터랙션**
- **상금 공개쇼 (예: 더지니어스, 유 퀴즈 ON THE BLOCK 점수 시퀀스)**

## 11. 다음 단계 체크리스트

디자인 시스템에서 이 문서로 시작해서 만들어야 할 것:

- [ ] 컬러 토큰 + 타이포 시스템
- [ ] 챗봇 캐릭터 디자인 + 5개 포즈 시안
- [ ] 6개 화면 상태별 와이어프레임 → 시안
- [ ] 핵심 5개 모멘트 모션 시안 (정지 이미지 + 모션 의도 설명)
- [ ] `PromptVersionCard` (프롬프트/결과 분리 패턴) 디테일
- [ ] `ScoreCard` 등장 모션 패턴
- [ ] `TotalScoreReveal` 카운트업 패턴
- [ ] 다크/라이트 결정 + 톤 스케일
